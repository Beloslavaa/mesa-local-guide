# Mesa · guía local

A digital zine of curated places in Madrid. The home screen shows a pile of
hand-illustrated leaflets that unfold in 3D when tapped, revealing the spots
inside. Each leaflet's map view uses real coordinates plotted on a
Thunderforest tile layer via Leaflet.js.

## How the app works

The 6 leaflets each represent a curated set of 5 Madrid spots. Tapping the
pile lifts the top leaflet and unfolds it as a 3D paper animation (CSS
`rotateY`/`rotateX` on pre-sliced image quadrants). Tapping the unfolded
leaflet opens a map view with real pins for the 5 spots. Tapping a list row
animates the map to that spot.

## "best now" feature (data-science layer)

The "best now" button surfaces the 3 spots most worth visiting *right now*
across all 6 leaflets, based on:

1. **Real busyness data** from Google's "popular times," fetched via SerpAPI.
2. **Unsupervised clustering** (k-means on each spot's weekday rhythm vector)
   to discover natural temporal categories — morning / afternoon / evening /
   late-night — without any manual labeling.
3. **A scoring function** that combines type-fit (how appropriate this kind of
   spot is for the current hour) and busy-fit (whether the spot is currently in
   its "moderately busy" sweet spot — not dead, not packed).

The clustering and data fetching are precomputed by the Python pipeline in
`data/`. The live app reads only the resulting JSON file — no API calls at
runtime, no Python on the server.

See `data/README.md` for how to regenerate the data.

## Tech

- **Frontend:** vanilla HTML/CSS/JS, no build step. Leaflet.js for the map,
  Thunderforest Transport for tiles, Google Fonts (Just Me Again Down Here).
- **Data:** Python 3 + SerpAPI + scikit-learn. Runs locally, ships a JSON file.
- **Hosting:** GitHub Pages (free, static).
